This repository covers **three categories** of design patterns:

### âœ… **Creational Patterns** (How objects are created)
These patterns deal with **object creation mechanisms**, making the process more flexible and reusable.

- `Singleton` â†’ Ensures only **one instance** of a class exists and provides a global access point.
- `Factory` â†’ Creates objects without specifying their **exact class**.
- `Builder` â†’ Constructs **complex objects step-by-step** while keeping the code readable.
- `Prototype` â†’ Creates new objects by **cloning** an existing object.

---

### âœ… **Structural Patterns** (How objects are composed)
These patterns focus on **how objects are structured** and how they work together efficiently.

- `Adapter` â†’ Allows incompatible interfaces to **work together** by acting as a bridge.
- `Composite` â†’ Treats **individual objects and groups of objects** uniformly.
- `Decorator` â†’ Dynamically **adds new behavior** to objects without modifying their code.
- `Proxy` â†’ Controls access to an object, adding an **extra layer** of functionality.

---

### âœ… **Behavioral Patterns** (How objects interact)
These patterns define **how objects communicate** and distribute responsibilities.

- `Template Method` â†’ Defines the **skeleton of an algorithm**, letting subclasses fill in specific steps.
- `Strategy` â†’ Allows swapping between **different algorithms** at runtime.
- `Observer` â†’ Notifies dependent objects **when a state changes** (like event listeners).
- `Iterator` â†’ Provides a way to **traverse elements** in a collection without exposing the structure.
- `Command` â†’ Encapsulates a **request as an object**, allowing for undoable operations.
- `Interpreter` â†’ Evaluates and processes **custom language or expression trees**.
